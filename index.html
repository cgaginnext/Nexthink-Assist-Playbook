How to prompt effectively

Follow this path to get the most from Nexthink Workspace — from first query to expert co-pilot.

Key principle: Always provide clear context and intent. Specify how you want the analysis done and what outcome you expect — then review, verify, and challenge the results.

Be specific

Name users, devices, apps, locations. Include time ranges and thresholds that matter.

Speak naturally

Ask like you'd ask a colleague. Nexthink Workspace translates intent into NQL automatically.

Break it down

For complex questions, go step by step to validate results and avoid wrong assumptions.

Iterate

Nexthink Workspace remembers the conversation. Build on previous answers to narrow your insight.

Frequently asked questions

Understand how Nexthink Assist works and how to use it effectively.

How does conversation sharing work?
When you share a conversation:

A copy is created — not the original.

This means:
  • Your original conversation remains private
  • The recipient gets an independent copy
  • They can continue the analysis on their own
👉 Any new interactions in the shared version:
  • Do not affect your original conversation
  • Create a new analysis path
Think of it like duplicating a document:
both versions start identical, then evolve independently.
Why does the same prompt sometimes produce different answers?
Nexthink Assist is powered by generative AI, which works very differently from traditional software.

It is not a predefined decision tree.
Instead of following fixed rules, it generates responses dynamically using probability.

Each response is built word by word by selecting the most likely next token from a range of possibilities.
Because several valid options exist at each step, small variations can naturally occur.

👉 Even a tiny difference early in a response can lead to a different final answer.

Think of it like asking several experts the same question:
they all give consistent insights, but phrase them differently or emphasize different aspects.
Is Nexthink Assist reliable if answers can vary?
Yes — the underlying analysis remains consistent, even if wording changes.

The variation mainly affects:
  • How the answer is phrased
  • The order of insights
It does not randomly invent results, but interprets data through slightly different angles.

👉 Best practice:
  • Iterate and refine your prompts
  • Validate key findings
  • Use follow-up questions to deepen analysis
How does conversation context impact results?
Assist remembers the context of your conversation.

This means:
  • Each question builds on previous answers
  • Implicit assumptions can carry over
👉 As a result, even the same prompt can produce different answers if:
  • The previous conversation differs
  • The analysis path evolves
How can I reduce variations and get more consistent results from the same prompt?

Key principle: The more structured and intentional your prompt is, the more consistent the results. Focus on clarity and precision — not length.

Because AI is probabilistic, small variations can appear between responses.

👉 To reduce this variability, structure your prompts by clearly defining:
  • Context (scope, population, timeframe)
  • Analysis method (how to approach the problem)
  • Objective (expected outcome)
  • Categorization rules
  • Output format

1. Always provide explicit context

Clearly define scope to avoid interpretation differences.

✅ Example:
Analyze Finance devices in France and Germany over the last 30 days. Focus on corporate Windows laptops used by active employees.

2. Specify the analysis method

Tell Assist how to think, not just what to answer.

✅ Example:
Method: 1. Identify affected devices 2. Group by device model 3. Compare vs previous 7 days

3. Clearly state the objective

Define what outcome you expect, especially for automation or reuse.

✅ Example:
Objective: Identify actionable issues that can be remediated via remote actions to reduce ticket volume.

4. Define categorization rules

Clear categorization ensures consistent interpretation.

✅ Example:
Categorize issues into: - Performance (CPU, memory, disk) - Application (crashes, freezes) - Network (latency, connectivity) - Configuration (policy, OS settings) Assign one category per issue.

5. Enforce a structured output

A fixed structure significantly reduces variability and improves usability.

✅ Example:
Output: - Table: Category | Devices | Root cause | Recommended action - 5-bullet executive summary

6. Add constraints for consistency

Explicit constraints limit interpretation drift.

✅ Example:
Constraints: - Do not infer missing data - Use consistent terminology - Focus only on actionable insights

7. Define advanced analysis expectations

For more reproducible results, specify how data should be processed and analyzed.

This includes:
  • How to categorize data
  • Which computations to perform
  • What outputs to generate
  • Which patterns to analyze

Categorization examples

Classify remote actions into: - Remediation - Deployment - Monitoring Use consistent keyword-based classification.

Computation examples

Calculate: - Total affected devices - % of total fleet - Avg issues per device - Trend vs previous 7 days

Output format examples

Include: - Table (structured results) - Line chart (trend over time) - Bar chart (top contributors)

Pattern analysis examples

Analyze patterns by: - Hour of the day - Day of the week - Remote vs office usage - Device model and OS version

Consistency tip: Compact, well-structured prompts are more reliable than long, descriptive ones. Focus on structure (sections, bullets, categories) rather than narrative text.

Automation-ready prompt
Analyze Windows devices in France, excluding servers (last 7 days). Method: 1. Devices with DEX < 50 2. Group by app + device model 3. Compare vs previous 7 days 4. Highlight anomalies Objective: Identify actionable issues for remote remediation Categorize: Performance / Application / Network / Configuration Output: - Table: Category | Devices | Root cause | Action - 5-bullet summary Constraints: - No assumptions if data missing - Consistent wording - Actionable insights only

  • Shared conversations
  • Repeatable analyses

It ensures outputs remain consistent, comparable, and actionable.
